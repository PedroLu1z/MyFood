# 🍔 Sistema de Banco de Dados para Delivery

Projeto desenvolvido durante o curso de **Banco de Dados para Data Science** ministrado pelo **SENAI Vila Mariana**.

O objetivo foi projetar e implementar um banco de dados relacional inspirado na estrutura de plataformas de delivery, utilizando o **iFood como referência de negócio**, aplicando conceitos de modelagem de dados e boas práticas de banco de dados.

> **Aviso:** Este projeto possui apenas fins educacionais e não possui qualquer vínculo com o iFood.

# 🎯 Objetivo

Desenvolver um banco de dados capaz de representar o funcionamento de uma plataforma de delivery, contemplando as principais entidades e seus relacionamentos.

O projeto foi desenvolvido para colocar em prática conhecimentos de:

- Modelagem Relacional
- Normalização
- Criação de Tabelas
- Chaves Primárias e Estrangeiras
- Integridade Referencial
- Consultas SQL
- Relacionamentos entre Entidades

---

# 🛠️ Tecnologias

- MySQL
- SQL

---

# 📚 Conceitos Aplicados

- Modelagem Conceitual
- Modelo Lógico
- Modelo Físico
- Normalização (1FN, 2FN e 3FN)
- Constraints
- Foreign Keys
- Joins
- Consultas SQL

---

# 📂 Estrutura do Banco

O banco contempla entidades como:

- Clientes
- Restaurantes
- Endereços
- Pedidos
- Itens do Pedido
- Produtos
- Categorias
- Pagamentos
- Entregadores
- Status do Pedido

Todas as tabelas foram modeladas visando representar um fluxo semelhante ao encontrado em plataformas de delivery.

# 💡 Aprendizados

Durante o desenvolvimento foi possível compreender melhor:

- Como grandes aplicações estruturam seus bancos de dados.
- A importância da normalização para evitar redundância.
- Como relacionar diversas entidades em um único sistema.
- A utilização de consultas SQL para extração de informações.
